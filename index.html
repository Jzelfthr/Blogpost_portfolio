The Pulse of Intelligence: Tuning the Learning Rate Within a Neural Mind

Exploring how learning rate shapes the journey of a neural network

1. Understanding Learning Rate

The learning rate is a hyperparameter that controls how much a neural network updates its weights during training. It determines the step size in gradient descent:

θ = θ - η * ∇L(θ)

Where θ are weights, η is the learning rate, and ∇L(θ) is the gradient of the loss.

The learning rate affects:

  • Convergence speed
  • Model stability
  • Generalization ability

2. Experimenting with Learning Rate

We use a small MLP on the MNIST dataset. Three learning rates were tested: 0.001, 0.01, 0.1.

# Sample PyTorch code
import torch
import torch.nn as nn
import torch.optim as optim

model = nn.Sequential(nn.Flatten(), nn.Linear(28*28, 128), nn.ReLU(), nn.Linear(128,10))
criterion = nn.CrossEntropyLoss()
optimizer = optim.SGD(model.parameters(), lr=0.01)  # change lr

Figure 1: Loss curves for different learning rates

3. Analysis and Insights

Observations:

  • Low learning rate (0.001): slow convergence, stable but longer training.
  • Medium learning rate (0.01): good balance of speed and stability.
  • High learning rate (0.1): fast but unstable, may overshoot minima.

Insight: Start with moderate learning rates and tune using learning rate scheduling or adaptive optimizers (Adam, RMSProp) for better performance.
